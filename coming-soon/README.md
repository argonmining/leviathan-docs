# Coming soon on testnet

Leviathan testnet is active today with the hosted explorer, browser wallet, and bridge monitor. The following pieces are planned to land **on testnet next week** (timeline may shift slightly as we integrate and test).

## Chrome extension wallet

A **Chrome extension wallet** is in final integration. It will offer the same programmable-layer account model as the hosted wallet, with keys and sync managed from the extension instead of only the explorer tab.

**Planned benefits:**

* Persistent wallet session while browsing other sites
* Cleaner signing flow for dapps that integrate with the extension
* Export/import compatibility with the same account formats pioneers use today

Until release, continue using [leviathandev.neptune.io/wallet](https://leviathandev.neptune.io/wallet) for create, fund, and send flows.

## Decentralized exchange (DEX)

A **testnet DEX** on the programmable layer is queued behind the extension wallet. It will let pioneers swap assets issued on Miden using the account-and-note execution model, with proofs verified on-chain.

Expect:

* Swap UI reachable from the explorer ecosystem
* Liquidity and pairs limited to testnet experimentation
* Documentation updated when contracts and addresses are deployed

## What is not changing

* Testnet-only scope — no mainnet promises in this doc
* Manual WXNT funding for pioneers via [request testnet funds](../getting-started/request-funds.md) until self-serve faucet policy changes
* Experimental bridge — monitor at [bridge](../bridge/README.md), exercise carefully

## Stay updated

Watch pioneer channels (Telegram/Discord) for the exact testnet deploy window. This page will be updated when the extension and DEX URLs go live.
